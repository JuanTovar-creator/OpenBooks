# 📚 OpenBooks - Biblioteca Virtual

## 📌 Descripción del Proyecto

OpenBooks es un prototipo web de un sistema de gestión de biblioteca virtual desarrollado en React.
Permite a los usuarios explorar, buscar y leer libros, mientras que los administradores pueden gestionar el contenido de la plataforma.

Este proyecto corresponde a un **prototipo no funcional**, enfocado en la interfaz gráfica y la navegación del sistema, sin conexión a un backend.

---

## 🛠 Tecnologías Utilizadas

* React
* JavaScript (ES6+)
* Vite
* React Router DOM
* CSS

---

## 🚀 Instalación y Ejecución

Sigue estos pasos para ejecutar el proyecto en tu entorno local:

1. Clonar el repositorio:

```bash
git clone https://github.com/JuanTovar-creator/OpenBooks.git
```

2. Acceder a la carpeta del proyecto:

```bash
cd OpenBooks
```

3. Instalar dependencias:

```bash
npm install
```

4. Ejecutar la aplicación:

```bash
npm run dev
```

5. Abrir en el navegador:

```
http://localhost:5173
```

## 👥 Roles del Sistema

### 👤 Usuario (Lector)

El usuario puede:

* Visualizar la biblioteca de libros
* Buscar libros
* Acceder a la lectura de libros
* Gestionar su perfil

### 🛠 Administrador

El administrador puede:

* Subir nuevos libros
* Editar libros existentes
* Eliminar libros
* Gestionar usuarios
* Visualizar reportes
* Supervisar el sistema

---

## 🖥️ Pantallas del Sistema

El sistema cuenta con múltiples pantallas, entre las principales:

* 🏠 Home (Inicio)
* 📚 Biblioteca de libros
* 📖 Vista de lectura de libros
* 👤 Perfil de usuario
* ⚙️ Panel de administración
* ➕ Subida de libros
* 📊 Reportes
* 👥 Gestión de usuarios
* ❌ Página de error (404)

---

## 🔄 Navegación del Sistema

La aplicación es completamente navegable mediante rutas, permitiendo:

* Acceder desde el inicio a la biblioteca
* Seleccionar un libro y visualizar su contenido
* Navegar entre las diferentes secciones del sistema
* Acceder a funcionalidades de administrador

---

## ⚠️ Consideraciones

* Este proyecto es un **prototipo no funcional** (no incluye backend).
* Los datos utilizados son simulados.
* Está diseñado para futuras integraciones con backend en C# (ASP.NET Core).
